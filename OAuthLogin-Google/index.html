發表於 2022-12-18 更新於 2023-01-15 分類於 筆記 ， 開發

寫在前面

  • Oauth 2.0 流程

    1. 客戶端提供其自身的信息，在資源所有者的允許下，向認證服務請求 Access Token。
    2. 認證服務驗證通過後，返回 Access Token 給客戶端。
    3. 客戶端使用 Access Token 向資源服務請求用戶數據。
    4. 資源服務驗證 Access Token 有效後，返回資源數據。

  • 角色
    客戶端 – 網頁/ App
    資源所有者 – 使用者
    認證服務 – 總管授權事務
    資源服務 – 提供訪問資源的api
